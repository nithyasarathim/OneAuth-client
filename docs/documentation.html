Product Overview

OneAuth Client is a React + Vite single-page application that acts as the shared authentication front door for multiple products. Instead of each product owning its own login and registration screens, this client centralizes the user journey around OneAuth.

The client is responsible for:

  • Registering new users through a guided verification funnel.
  • Logging users in through the OneAuth session API.
  • Protecting private UI routes through live session validation.
  • Redirecting authenticated users through the SSO authorization experience.
  • Managing profile, avatar, and password workflows from one account portal.

src/pages/ directory

The src/pages/ directory is organized by product workflow instead of by technical layer. Each major route owns its UI, validation, API calls, and local types. This keeps authentication flows easy to reason about because the code needed for a screen lives close to the screen itself.

Area Responsibility
auth/login/ Login, password recovery, login validation, and session creation calls.
auth/register/ Email verification, OTP validation, password setup, and account creation.
profile/ Account profile, avatar upload, profile editing, and password changes.
singleSignOn/ SSO authorization screen and API bridge for client applications.
globalComponents/ Route guards and app-wide request/session behavior.
utils/ Environment mapping, Axios configuration, and shared API error handling.

Modular api/ folders

Each feature has its own api/ folder. This pattern avoids a single oversized API file and keeps the service contract near the UI that consumes it.

  • src/pages/auth/login/api/login.api.tsx
  • src/pages/auth/register/api/register.api.tsx
  • src/pages/profile/api/profile.api.tsx
  • src/pages/profile/api/upload.api.tsx
  • src/pages/singleSignOn/api/sso.api.ts

Routing Logic

ProtectedRoute.tsx

ProtectedRoute guards authenticated pages such as /, profile screens, and /sso.

Incoming Request
↓
GET /auth/session/validate
Called through configured Axios client
↓
Loading State
Page stays blank while validation is pending
↓
Session Valid?
↓
Allow Render
Or redirect to /auth/login with ?redirect= param

This keeps protected pages from rendering sensitive UI before the session is verified. The original path is preserved in the redirect query parameter so the user can return after login.

GuestRoute.tsx

GuestRoute protects public authentication screens such as /auth/login and /auth/register.

Public Page Request
↓
GET /auth/session/validate
↓
No Session?
Allow the public page to render
↓
Has Session → /
Redirect authenticated users away from login/register

GlobalAPIWatcher.tsx

GlobalAPIWatcher registers a global Axios response interceptor for app-wide session failures. When any response returns 401 Unauthorized, the watcher navigates the user to /auth/login and replaces the current history entry. This creates one centralized escape hatch for expired or invalid sessions instead of handling that redirect separately in every component.

SSO Role in the Ecosystem

OneAuth Client is the place where users prove who they are. Product applications redirect users to this portal when authentication is required. After the API validates the session and client metadata, OneAuth can issue an authorization code that the product backend exchanges for a JWT access token.

Product App
→
OneAuth Client
↓
OneAuth API
Session validated, client metadata checked
↓
HTTP-only Cookie
Session persisted in Redis-backed cookie
↓
Auth Code → Product
Product backend exchanges code for JWT

Environment Variables

Create a .env file in the client project root. All variables are read in src/pages/utils/env.ts and passed into the shared Axios instance in src/pages/utils/axios.ts.

VITE_AUTH_SERVER_API_URL=http://localhost:8000
VITE_REQUEST_TIME_OUT=10000
Variable Default / Current Value Description
VITE_AUTH_SERVER_API_URL http://localhost:8000 Base URL for the OneAuth API. Used by the shared Axios client.
VITE_REQUEST_TIME_OUT 10000 ms Request timeout in milliseconds. Current intended value is 10 seconds.

Installation & Usage

Prerequisites

  • Node.js 20 or newer is recommended.
  • OneAuth API running locally or remotely.
  • A valid .env file in the client root.

Install dependencies

npm install

Run the development server

npm run dev

Vite will print a local URL, usually:

http://localhost:5173

Build for production

npm run build

Preview the production build

npm run preview

Available Scripts

Command Description
npm run dev Starts the Vite development server.
npm run build Type-checks the app and creates a production build.
npm run lint Runs ESLint across the project.
npm run preview Serves the production build locally for verification.

Repository Notes

  • The app uses withCredentials: true, so the API must enable CORS credentials for the client domain.
  • Route guards depend on /auth/session/validate; keep that endpoint stable.
  • API request timeouts are controlled by VITE_REQUEST_TIME_OUT.
  • Feature folders should keep their API functions, validators, and types close to the page they serve.
